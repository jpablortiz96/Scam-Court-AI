# 📇 Model Card — Scam Court AI

## Model Details

- **Project:** Scam Court AI
- **Developers:** Scam Court AI Team (Hackathon project)
- **Hackathon:** Hugging Face Build Small
- **License:** MIT
- **Model type:** Multi-agent safety classifier with heuristic and planned small-model backends
- **Language(s):** English (heuristic), extensible to multilingual via backend swap
- **Library:** `transformers` (planned), `gradio` (UI)
- **Total parameters (planned):** ≤ 32B
- **Hardware requirement (MVP):** CPU only
- **Hardware requirement (full):** 8GB+ RAM for 3B model; 16GB+ for vision models

## Model Description

Scam Court AI is a protective multi-agent system that analyzes suspicious messages (SMS, WhatsApp, email, marketplace chats) and renders a structured courtroom verdict. The system is designed to:

1. **Detect** red flags via heuristic rules or small-model reasoning.
2. **Prosecute** by explaining manipulation tactics.
3. **Defend** by stress-testing legitimacy.
4. **Judge** with a calibrated risk score (0–100) and verdict.
5. **Protect** by generating safe replies and actionable next steps.

The current MVP uses a deterministic heuristic engine. Future iterations will load:
- `HuggingFaceTB/SmolLM3-3B` as the reasoning core (Tiny Titan track).
- `openbmb/MiniCPM-V-4_5` for screenshot analysis (OpenBMB track).
- NVIDIA Nemotron small models for agentic debate (optional).

## Intended Use

- **Primary:** Consumer protection against SMS, email, and messaging scams.
- **Secondary:** Educational tool for digital literacy and social-engineering awareness.
- **Out of scope:** Legal advice, malware analysis, deepfake detection.

## How to Use

```python
from courtroom import CourtroomEngine

engine = CourtroomEngine()
report = engine.analyze("Your bank account has been suspended...")
print(report.risk_score)        # 0-100
print(report.judge_verdict)     # SCAM / SUSPICIOUS / CAUTION / LIKELY SAFE
print(report.clerk_safe_reply)  # actionable guidance
```

Run the UI:
```bash
python app.py
```

## Training Data

- **Heuristic engine:** Hand-crafted regex patterns derived from public scam-reporting datasets (FTC Consumer Sentinel, ACMA Scamwatch) and academic literature on phishing linguistics.
- **No fine-tuning data** is used in the MVP.
- Future model-backed versions may use:
  - Public phishing datasets (Enron + modified, SpamAssassin).
  - Synthetic adversarial examples generated by larger models.
  - Human-verified scam reports (with PII stripped).

## Limitations

- Heuristic engine cannot generalize to novel scam formats not covered by regex rules.
- Risk score is a weighted heuristic, not a calibrated probability.
- English-first; non-English scams may be under-detected.
- No image understanding in MVP (planned for Phase 3).
- Cannot verify external URLs in real time.

## Ethical Considerations

- **Privacy:** All analysis runs locally. No message text is transmitted to third-party APIs.
- **Bias:** Heuristic patterns may over-flag legitimate messages from non-native English speakers or informal communicators. The defender persona is designed to mitigate false positives.
- **Safety:** The system does not block messages; it informs the user. Users retain full agency.
- **Transparency:** Every verdict includes rationale. The JSON export enables auditability.
