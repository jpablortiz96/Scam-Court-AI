# Scam Court AI Demo Plan

## Target Length

75 seconds, with a usable range of 60-90 seconds.

## Recording Setup

- Start on Shield Mode with the premium UI visible.
- Use a legible synthetic FedEx screenshot.
- Confirm the Space shows `Vision: minicpm_v`.
- Keep the evaluation report open in a second tab for the proof shot.
- Avoid showing environment settings, tokens, browser bookmarks, or private tabs.

## Script and Shot List

### 0-7 seconds: Opening Hook

**Visual:** FedEx-style delivery screenshot beside the live Space.

**Voiceover:**  
"A scam does not arrive when someone is ready to investigate. It arrives when
they are rushed and one click away from harm."

### 7-20 seconds: Vision Witness

**Action:** Upload the suspicious FedEx screenshot in Shield Mode and analyze.

**Visual proof:** MiniCPM-V Vision Witness shows extracted delivery text and the
action link.

**Voiceover:**  
"Scam Court AI uses OpenBMB MiniCPM-V-4 as a Vision Witness. It reads the
screenshot and sends the visible evidence into the same safety pipeline."

### 20-31 seconds: 3-Second Shield

**Visual:** `VERIFY FIRST`, risk score, and the instruction not to click.

**Voiceover:**  
"The first answer is not a paragraph. It is a safe action: verify first. Open
the official carrier site yourself; never use the link in the message."

### 31-46 seconds: Court Explanation

**Action:** Switch to Court Mode and briefly reveal the role cards.

**Visual:** Detective evidence, Prosecutor argument, Defender uncertainty,
Judge verdict, Safety Clerk next steps.

**Voiceover:**  
"Then the court explains the decision. The Detective finds the package action
and link. The Prosecutor explains the tactic. The Defender tests a legitimate
interpretation. The Judge and Safety Clerk turn that into a cautious action."

### 46-56 seconds: Active-Call Rescue

**Action:** Open Suspicious Call and select money, code, urgency, or family-new-number signals.

**Voiceover:**  
"If the scam is happening on a call, five fast questions help the user stop
before sharing money or a code."

### 56-64 seconds: Companion Preview

**Action:** Show the WhatsApp, SMS, and marketplace Companion Preview.

**Voiceover:**  
"The Companion Preview shows where this goes next: bring the Shield verdict to
the selected message instead of making the user leave the conversation."

### 64-72 seconds: Evaluation Proof

**Visual:** Evaluation report summary: 60 cases, 0 false low-risk, 0 safety failures.

**Voiceover:**  
"The safety policy is measurable: 60 synthetic regression cases across ten
scam categories, with zero false low-visible-risk results in the current baseline."

### 72-78 seconds: Closing

**Visual:** Hero and live Space URL.

**Voiceover:**  
"Scam Court AI puts suspicious messages on trial, but gives families the shield first."

## On-Screen Proof Checklist

- Live Hugging Face Space URL
- MiniCPM-V model name
- Extracted screenshot text
- `VERIFY FIRST` verdict
- Court role cards
- Suspicious Call interaction
- Companion Preview
- Evaluation metrics
- UI version/build marker

## Social CTA

"Try the live Space. Upload a synthetic suspicious message, inspect the agent
trace, and review the safety evaluation."

## Backup Plan

If ZeroGPU cold start is slow:

1. Start the vision request before recording.
2. Keep a completed screenshot analysis in another browser tab.
3. Record the live upload briefly, then cut to the completed result.
4. Do not imply that a preloaded result completed instantly.
